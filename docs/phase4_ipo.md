# Phase 4 — 개인 애널리스트 확장 IPO 명세

> **목표**(기획안 4장): "뉴스를 잘 요약해 주는 도구"를 넘어 "업계 흐름을 먼저 읽어주는 개인 애널리스트"로 진화.
> 기존 Step 1~6 파이프라인 위에 얹는 확장이며 코어 로직 변경 없이 추가한다. 공통 기반은 @docs/phase1_ipo.md 참조.
> **권장 도입 순서**: 4-1 → 4-2 → 4-3 (기획안 4장 "권장 순서").

## 4-1. 이상 신호 감지 & 즉시 속보 알림

- **목적**: 하루 1회(08:30) 브리핑 구조를 유지하되, 특정 이슈가 평소 대비 비정상 급증하면 정규 브리핑을 기다리지 않고 즉시 알림. "속보를 다음 날 아침까지 놓치는" 리스크 해결
- **트리거 조건**: 등록 소스 전체 기준, 특정 기업명 + 위험 키워드("규제"·"화재"·"셧다운"·"리콜" 등)의 최근 1시간 언급 건수가 최근 7일 같은 시간대 평균 대비 임계치(예: 3배) 초과
- **Input**: Step 1 수집 결과를 시간 단위로 재사용, 소스별 키워드 언급 빈도 이동평균 테이블(신규 저장)
- **Process**: ① 매시 정각 최근 1시간 신규 기사 수집 → ② 키워드·기업별 빈도 집계 → ③ 임계치 초과 시 Gemini API로 사실관계 확인 및 `[확정]`/`[관측]` 판정 → ④ 확정 속보만 즉시 이메일, 나머지는 정규 08:30 브리핑에 포함
- **Output**: 즉시 속보 이메일 + 이동평균 테이블 갱신
- **알림 억제**: 같은 이슈로 이미 속보를 보냈으면 24시간 내 재발송 금지, 정규 브리핑에서만 갱신
- **기술**: 간단한 통계 로직(Python) + Gemini API 최종 판별(진짜 속보 vs 재탕 보도)

```json
// data/signals/hourly_freq.json (키워드·기업별 이동평균)
[ { "entity": "삼성전자", "keyword": "화재", "hour": "ISO8601",
    "count_1h": 12, "avg_7d_same_hour": 3.1, "ratio": 3.87 } ]
```

## 4-2. 이슈 지식그래프 (Issue Knowledge Graph)

- **목적**: 단발성 기사 요약을 넘어 동일 이슈를 시간순으로 자동 연결해 "언제 시작돼 어떻게 전개되는가"를 보여줌. 기획안 ④ 맥락화·해석 단계의 사람 부담을 실제로 줄임
- **산출물**: 브리핑 하단 "진행 중인 이슈 타임라인" 카드 (예: "HBM 증설 이슈 — 3주간 5개 기사 누적, 최초 보도 6/15")
- **Input**: Step 4 요약 결과의 날짜별 아카이브, 이슈 ID·연결관계를 저장하는 신규 테이블
- **Process**: ① 매일 요약된 기사마다 기존 이슈 목록과 매칭 시도 → ② 매칭되면 같은 이슈 ID로 연결, 없으면 신규 이슈 생성 → ③ 브리핑 조립 시 진행 중 이슈를 타임라인 카드로 삽입 → ④ 이슈가 3일 이상 이어지면 Gemini API가 "경과 요약" 한 문단 추가 생성
- **기술**: Gemini API 의미 유사도 매칭(후속 보도 판정) + `company_aliases.yaml` 기업명 정규화
- **Output**: `data/issues/graph.json` — 이슈 노드 + 기사 연결
- **확장 여지**: "이 이슈, 예전에도 있었나?" 질문에 아카이브를 검색해 답하는 RAG 챗봇으로 발전 가능

```json
// data/issues/graph.json
[ { "issue_id": "hbm-expansion", "title": "HBM 증설",
    "first_seen": "2026-06-15", "article_ids": ["...", "..."],
    "timeline": [ { "date": "2026-06-15", "summary": "..." } ],
    "progress_summary": "3주간 5개 기사 누적..." } ]
```

## 4-3. 경쟁 구도 레이더 (Competitive Radar)

- **목적**: 관심 기업(삼성전자·SK하이닉스·TSMC·인텔·엔비디아 등)을 지정하면 주간 단위로 "누가 뉴스에 가장 많이, 어떤 맥락으로 언급됐는지"를 비교하는 미니 리포트 자동 생성
- **산출물**: 주 1회(예: 매주 월요일 브리핑 첨부) 기업별 언급량 막대그래프 + 긍정/부정/중립 톤 비율 + 해당 주 최다 언급 이슈 3건 요약
- **Input**: Step 3 분류 단계에서 이미 태깅된 기사의 기업명·카테고리 정보를 주간 단위로 누적 집계
- **Process**: ① 매일 분류된 기사에 기업 태그·톤 라벨 추가 저장 → ② 주 1회 배치로 지난 7일치 집계 → ③ Gemini API로 "이번 주 이슈의 중심" 한 문단 해설 생성 → ④ 정규 브리핑 템플릿에 주간 섹션으로 삽입
- **기술**: Python 집계·시각화(matplotlib 등 무료 라이브러리) + Gemini API 톤(긍정/부정/중립) 판정 및 이슈 요약
- **Output**: 주간 리포트 섹션 + `data/radar/weekly-YYYY-Www.json`
- **확장 여지**: 월간 누적 시 "3개월간 경쟁사 뉴스 점유율 변화" 같은 회고형 인사이트로 발전 (Step 6 아카이브 재활용)

```json
// data/radar/weekly-YYYY-Www.json
{ "week": "2026-W28",
  "mentions": { "삼성전자": 42, "SK하이닉스": 31, "TSMC": 28 },
  "tone": { "삼성전자": { "pos": 0.5, "neg": 0.2, "neu": 0.3 } },
  "top_issues": ["...", "...", "..."] }
```
