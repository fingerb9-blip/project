# Phase 5 — 개인 애널리스트 확장 II (국내 주가 연동 + 언급량 트렌드) IPO 명세

> Phase 4(개인 애널리스트 확장 — 경쟁 구도 레이더)에 이어지는 확장이며, 기존 Step 1~6
> 파이프라인 위에 얹는다. 코어 로직 변경 없이 추가한다. 공통 기반은 @docs/phase1_ipo.md 참조.
> v2 변경점: 주가 연동을 **국내 종목 전용**으로 한정, 기업 언급량 트렌드에 **기술 키워드(HBM, EUV 등)
> 언급량 트렌드**를 추가했다.

## 0. 목표와 전제

| 항목 | 내용 |
|---|---|
| 목표 A | 국내 관련 기업(삼성전자, SK하이닉스 등)의 당일 주가 변동을 뉴스와 함께 보여줘 맥락 강화 |
| 목표 B | 기사에서 언급된 기업을 태깅·집계해 언급량 추이를 시각화 |
| 목표 C | HBM, EUV, 2나노 공정 등 기술 키워드의 언급량 추이를 별도로 집계해, 특정 기술 트렌드가 뜨고 있는지 포착 |
| 전제 | Phase 1의 `src/`, `config/` 모듈이 정상 동작 중, 대시보드가 `data/classified` 또는 `data/summarized` 결과를 읽어 렌더링하는 구조 |
| 변경 범위 | Step 3(분류) 프롬프트 확장, 신규 Step(주가 수집·기업/키워드 집계) 추가, config 신규 파일, 대시보드 UI 확장 |
| 변경 없음 | Step 1(수집), Step 2(중복 제거) 로직, Phase 2/3/4의 스케줄링·워치리스트·레이더 로직 |
| **범위 제외** | 해외 종목(TSMC, ASML, Nvidia 등) 주가 연동 — 국내 종목만 다룬다 |

## 1. 기능 A: 국내 주가 연동

### 목적
특정 뉴스가 나온 날 관련 국내 기업의 주가가 어떻게 반응했는지 함께 보여줘, 기사만으로는 알기 어려운 시장의 해석을 덧붙인다.

### 데이터 소스

| 대상 | 추천 소스 | 비고 |
|---|---|---|
| 국내 종목(삼성전자, SK하이닉스, DB하이텍 등) | `pykrx` | API 키 불필요, 무료. KRX·네이버 데이터를 스크래핑하는 방식이라 참고용으로만 사용, 무분별한 호출은 자제 |
| (대안) | 한국투자증권 Open API | 무료 발급, 실전/모의투자 계정으로 국내주식 현재가·일봉 조회 가능. LLM 연동용 샘플 코드도 공개돼 있어 Gemini 파이프라인에 붙이기 편함 |

두 소스 모두 비용 없이 시작 가능하며, 우선 `pykrx`로 MVP를 만들고 안정성이 더 필요해지면 한국투자증권 API로 교체하는 순서를 권장한다.

### 신규 모듈: `step_stock_price.py`

- **Input**: `config/watch_tickers.yaml` (국내 기업명–종목코드 매핑)
- **Process**:
  1. 매일 배치(Phase 2 GitHub Actions cron 이후 실행)로 등록된 국내 종목의 전일 종가·등락률 조회
  2. 조회 실패 시 직전 값 유지 + 로그 기록 (알림은 발송하지 않음, Step 6 조립에 영향 없도록)
- **Output**: `data/stock/YYYY-MM-DD.json`

```json
{
  "date": "2026-07-09",
  "tickers": [
    { "ticker": "005930", "name": "삼성전자", "close": 92300, "change_pct": 1.8 },
    { "ticker": "000660", "name": "SK하이닉스", "close": 215000, "change_pct": 2.4 }
  ]
}
```

### `config/watch_tickers.yaml` 예시 (국내 종목만)

```yaml
tickers:
  - name: "삼성전자"
    ticker: "005930"
  - name: "SK하이닉스"
    ticker: "000660"
  - name: "DB하이텍"
    ticker: "000990"
  - name: "한미반도체"
    ticker: "042700"
```

### 뉴스–주가 연결 로직

- Step 3(분류)에서 추출되는 `companies` 필드(기능 B와 공유)를 기준으로, 기사에 언급된 국내 기업의 당일 `change_pct`를 매칭
- 매칭 결과는 조립(Step 5)에 필드 추가: `related_stock: [{name, change_pct}]`
- 해외 기업이 `companies`에 잡혀도 주가 매칭은 시도하지 않고 건너뜀 (watch_tickers.yaml에 없는 이름은 무시)

### 대시보드 반영 사항

| UI 요소 | 내용 |
|---|---|
| 기사 카드 뱃지 | 관련 국내 기업명 옆에 당일 등락률(%) 표시, 상승/하락 색상 구분 |
| 종목 상세 뷰 | 최근 N일 가격 추이 차트 + 같은 기간 관련 뉴스를 타임라인으로 오버레이 |

### 에러·알림 규칙 추가

| 상황 | 처리 |
|---|---|
| 특정 종목 조회 실패 | 직전 값 유지, 뱃지에 "시세 지연" 표시, 파이프라인 계속 진행 |
| pykrx 소스 자체 장애 | 그날은 주가 뱃지 전체 숨김, 기존 알림 로직(Phase 1)에 영향 없음 |

## 2. 기능 B: 기업 언급량 트렌드

### 목적
기사에서 언급되는 기업명을 집계해 날짜별 추이로 보여줌으로써, 특정 기업 관련 이슈가 최근 얼마나 자주 다뤄지는지 파악하고, Phase 3의 이상 신호 감지(언급량 급증)와 연계한다.

### Step 3(분류) 프롬프트 확장

- 기존 분류 출력 스키마에 `companies: string[]` 필드 추가
- Gemini API 프롬프트에 "기사 본문에서 언급된 반도체 관련 기업명을 표준명으로 추출" 지시 추가
- 표준명 통일을 위해 `config/companies.yaml`의 별칭 매핑을 함께 투입(참고 데이터 파일 방식, 기존 4종 yaml과 동일한 패턴)

### `config/companies.yaml` 예시

```yaml
companies:
  - canonical: "삼성전자"
    aliases: ["삼성", "삼성전자 파운드리", "Samsung Electronics"]
  - canonical: "SK하이닉스"
    aliases: ["SK Hynix", "하이닉스"]
  - canonical: "TSMC"
    aliases: ["대만 TSMC", "Taiwan Semiconductor"]
```

> 해외 기업명도 표준화 대상에는 포함한다 — 언급량 트렌드 자체는 국내외 기업 모두 다루되, 주가 매칭(기능 A)만 국내 종목으로 한정된다.

## 3. 기능 C: 기술 키워드 언급량 트렌드

### 목적
HBM, EUV, 2나노 공정, 첨단 패키징(CoWoS 등)처럼 기업명이 아닌 **기술 용어**의 언급 빈도를 따로 추적해, 어떤 기술 트렌드가 최근 뜨고 있는지 조기에 포착한다. 기업 언급량(기능 B)과는 별도 축으로 관리한다.

### Step 3(분류) 프롬프트 확장

- 기존 스키마에 `keywords: string[]` 필드 추가
- Gemini API 프롬프트에 "기사 본문에서 언급된 반도체 기술 키워드를 사전에 정의된 목록 기준으로 추출" 지시 추가
- `config/tech_keywords.yaml`을 참고 데이터로 함께 투입해, 사전에 등록된 키워드 위주로 추출하도록 유도(자유 추출 시 노이즈가 커지는 것을 방지)

### `config/tech_keywords.yaml` 예시

```yaml
keywords:
  - canonical: "HBM"
    aliases: ["고대역폭메모리", "HBM3", "HBM3E", "HBM4"]
  - canonical: "EUV"
    aliases: ["극자외선", "EUV 노광"]
  - canonical: "첨단 패키징"
    aliases: ["CoWoS", "2.5D 패키징", "칩렛"]
  - canonical: "2나노 공정"
    aliases: ["2nm", "GAA"]
```

### 신규 모듈: `step_mention_trend.py` (기능 B와 통합 운영)

- **Input**: `data/classified/YYYY-MM-DD.json` (companies, keywords 필드 포함된 분류 결과), 누적 이력
- **Process**:
  1. 일별 기업별 / 키워드별 언급 건수를 각각 집계 (같은 로직, 대상 사전만 다름)
  2. 7일/30일 이동 평균과 비교해 급증 여부 플래그 생성 (Phase 3 이상 신호 감지 로직과 동일한 통계 방식 재사용)
- **Output**: `data/trends/mention_trend.json`

```json
{
  "date": "2026-07-09",
  "companies": [
    { "name": "삼성전자", "count": 5, "is_spike": false }
  ],
  "keywords": [
    { "name": "HBM", "count": 8, "is_spike": true },
    { "name": "EUV", "count": 2, "is_spike": false }
  ]
}
```

### 대시보드 반영 사항

| UI 요소 | 내용 |
|---|---|
| 언급량 트렌드 차트 | 기업 탭 / 키워드 탭을 나눠서 각각 최근 N일 언급 건수 라인 차트 |
| 급증 하이라이트 | 이동 평균 대비 급증 구간을 차트에 표시, 클릭 시 관련 기사 목록 연결 |
| 키워드 랭킹 | 최근 7일 기준 언급량 상위 키워드 순위를 사이드바에 노출 |

---

## 4. 기능 간 연결점

- 기업 언급량 급증 + 국내 주가 변동을 같은 화면(종목 상세 뷰)에 겹쳐 보여주면 "이슈가 실제로 시장에 영향을 줬는지" 바로 확인 가능
- 키워드 급증(예: HBM 언급 급증)이 특정 기업 언급량 급증과 같은 시기에 나타나면, 그 기업이 해당 기술 트렌드의 중심이라는 신호로 해석 가능 — 대시보드에서 두 트렌드를 나란히 배치하는 것을 권장

## 5. 콜드 스타트 조건

> v3 변경점: 주가 뱃지(기능 A)는 누적 이력이 없어도 당일 값만으로 바로 계산되므로 콜드 스타트
> 대상에서 제외한다 — 항상 노출. 콜드 스타트는 이동 평균·급증 판정처럼 누적 이력이 필요한
> 언급량 트렌드(기능 B·C)에만 적용한다.

| 데이터 누적 기간 | 주가 뱃지(기능 A) | 언급량 트렌드(기능 B·C) |
|---|---|---|
| 14일 미만 | 항상 노출 | 비활성화 |
| 14~21일 | 항상 노출 | "참고용" 라벨 표시 |
| 21일 이후 | 항상 노출 | 정식 운영 |

## 6. 구현 지침

- 기존 Step 1~6, Phase 2/3/4 로직은 건드리지 않는다.
- `step3_classify.py` 프롬프트에 `companies`, `keywords` 필드만 추가한다.
- `src/step_stock_price.py`(국내 종목 전용, pykrx 기반), `src/step_mention_trend.py`(기업+키워드 통합)를 신규 생성한다.
- `config/watch_tickers.yaml`(국내 종목만), `config/companies.yaml`, `config/tech_keywords.yaml`을 신규 생성한다.
- 5장 콜드 스타트 조건부터 확인한다. 주가 뱃지(기능 A)는 콜드 스타트와 무관하게 항상 노출하고, 언급량 트렌드(기능 B·C)만 데이터 14일 미만이면 비활성화 상태로 먼저 만든다.
