# 이메일 뉴스레터 발송 — 설계 (무료, 지인 대상 MVP)

> 작성일: 2026-07-10 · 상태: 설계 승인 대기
> 기존 Step 0~6 파이프라인 위에 얹는 확장. 코어 로직 변경 없음.

## 1. 목표와 범위

| 항목 | 내용 |
|---|---|
| 목표 | 매일 파이프라인이 브리핑을 발행한 뒤, 명단의 구독자에게 그날 브리핑을 이메일로 발송한다. |
| 성격 | **무료** 이메일 뉴스레터. 결제·회원가입·DB·로그인 없음. 수신 동의한 지인 소수(10명대) 대상. |
| 핵심 가치 | 대시보드를 열지 않아도 매일 아침 정리된 브리핑을 편하게 메일로 받아본다. |
| 발송 인프라 | 기존 실패 알림에 쓰는 **Gmail SMTP 그대로 재사용** (하루 10통대는 한도 내). |

### 범위 — 포함
- 구독자 명단 로드(시크릿 기반)
- 그날 대시보드 HTML을 **CSS 인라인한 자립형 HTML**로 만들어 첨부
- 메일 본문에 "오늘의 핵심" **상위 3개** 요약 + 대시보드 링크 + 구독취소 안내
- 기존 SMTP로 **1인씩 개별 발송**(구독자 이메일 상호 노출 방지)
- 하루 1회 발송(중복 방지), 발송 실패가 파이프라인을 중단시키지 않음

### 범위 — 제외 (무료라서 불필요)
- 결제/정기결제, 회원가입/인증, 구독자 DB, 통신판매업 관련 로직, Gemini 유료 전환
- 웹 가입 폼(신규 가입은 수동으로 명단 시크릿에 추가)

## 2. 아키텍처

```
Step 0~6 (기존, 변경 없음)
   └─ Step 6 (대시보드 발행) 성공
        └─ [신규] Step 7: 이메일 뉴스레터 발송  ← 실패해도 파이프라인 성공 유지
```

- `main.py`에서 Step 6이 성공한 경우에만 Step 7을 호출한다.
- Step 7은 `try/except`로 감싸 어떤 실패도 파이프라인 종료 코드에 영향을 주지 않는다(브리핑 자체는 이미 발행·배포됨).

## 3. 신규 모듈: `src/step7_subscriber_email.py`

### Input
- 구독자 명단: 환경변수 `SUBSCRIBERS` (콤마 구분 이메일). GitHub Actions Secret으로 주입, 로컬은 `.env`.
- 그날 요약 데이터: `data/summarized/{today}.json` (본문 상위 3개 구성용)
- 그날 대시보드 HTML: `data/dashboard/{today}.html` + `data/dashboard/style.css` (자립형 첨부 생성용)
- 발송 상태: `data/state/newsletter_state.json` (중복 발송 방지)
- SMTP 자격: 기존 `SMTP_HOST/PORT/USER/PASSWORD` 환경변수

### Process
1. **중복 방지 가드**: `newsletter_state.json`의 `last_sent_date == today`이고 `FORCE_RESEND`가 아니면 즉시 종료(스킵).
2. **명단 로드** `load_subscribers()`: `SUBSCRIBERS`를 콤마로 분리, 공백 제거, 기본 형식(`@` 포함) 검증, 중복 제거. 비어 있으면 로그만 남기고 종료.
3. **자립형 HTML 생성** `build_standalone_html(today)`: `{today}.html`의 `<link rel="stylesheet" href="style.css">`를 `<style>…</style>`(인라인된 CSS)로 치환. CSS는 `step5_assemble._DASHBOARD_CSS`를 사용.
4. **본문 생성** `build_email_body(today)`: 요약 데이터에서 "오늘의 핵심" 상위 3개(`step5_assemble.select_highlights` 재사용, `[:3]`)를 뽑아 각 항목을 `제목 + [확정]/[관측] + 한 줄 요약`으로 렌더. 하단에 대시보드 링크와 **구독취소 안내**("구독을 원치 않으시면 이 메일에 회신해 주세요").
5. **발송** `send_to_subscribers(...)`: 각 구독자에게 개별 메일(HTML 본문 + 자립형 HTML 첨부) 발송. 개별 발송 실패는 건별로 로그하고 다음 구독자로 계속.
6. **상태 저장**: 한 명이라도 성공하면 `last_sent_date = today` 기록.

### Output
- 발송 성공/실패 건수 로그
- `data/state/newsletter_state.json` 갱신 (`{"last_sent_date": "YYYY-MM-DD", "sent_count": N}`)

## 4. 이메일 형식

- **제목**: `[반도체 브리핑] {today} 오늘의 핵심`
- **본문(HTML)**:
  - "오늘의 핵심" 상위 3개 — 제목 + `[확정]`/`[관측]` 태그 + 한 줄 요약(길면 잘라냄)
  - "전체 브리핑은 첨부 파일 또는 대시보드에서 보실 수 있습니다" + 대시보드 URL 링크
  - 구독취소 안내 한 줄
- **첨부**: `반도체브리핑_{today}.html` (CSS 인라인된 자립형, 브라우저에서 바로 열림)

## 5. 설정·보안

- ⚠️ **이 저장소는 공개**다. 구독자 이메일(PII)을 **절대 커밋하지 않는다.** 명단은 GitHub Actions Secret `SUBSCRIBERS`로만 주입하고, 로컬 테스트는 `.env`(gitignore됨)를 쓴다.
- 신규 구독은 **수동**: 수신 동의한 사람의 이메일을 시크릿 목록에 추가한다(임의 수집 금지).
- 발송 계정은 기존 SMTP 자격을 재사용한다.

## 6. 에러 처리·엣지

| 상황 | 처리 |
|---|---|
| Step 6 실패(브리핑 미발행) | Step 7 호출 안 함 |
| 오늘 이미 발송함 | 스킵(중복 방지 가드), `FORCE_RESEND=true`면 재발송 |
| 명단 비어 있음 | 로그만, 정상 종료 |
| 특정 구독자 SMTP 실패 | 건별 로그 후 다음 구독자로 계속 |
| 전체 SMTP 장애 | 로그만, **파이프라인은 성공 유지**(브리핑·대시보드는 이미 발행됨) |
| 뉴스 없는 날 | 그대로 발송(빈 핵심이면 그 사실을 본문에 안내) |

## 7. 테스트 (SMTP는 mock)

- `load_subscribers`: 콤마 분리·공백·중복·형식 검증, 빈 값 처리
- `build_standalone_html`: `<link>`가 인라인 `<style>`로 치환되고 CSS가 포함됨
- `build_email_body`: 상위 3개만 포함, 제목·태그·요약·대시보드 링크·구독취소 안내 포함
- 중복 발송 가드: 같은 날 두 번 호출 시 두 번째는 스킵, `FORCE_RESEND`면 재발송
- 개별 발송 실패가 다른 구독자 발송과 파이프라인을 막지 않음
- Step 6 실패 시 Step 7 미호출 (main 연동)

## 8. 법적·윤리 참고 (무료·소규모 전제)

무료·지인 대상이라 결제/통신판매 이슈는 없다. 다만 유지할 것:
- **수신 동의 + 구독취소 수단**(정보통신망법 취지) — 메일에 취소 안내 포함, 동의한 사람만 명단 추가
- **저작권 예의** — 요약은 짧게, 항상 원문 링크 포함("링크+코멘트" 성격 유지)
- 나중에 유료화하거나 규모를 키우면: 통신판매업 신고, 콘텐츠 라이선스, Gemini 유료 전환, 전용 이메일 발송 서비스 재검토
