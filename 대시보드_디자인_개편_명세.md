# 반도체 뉴스 브리핑 대시보드 — 디자인 개편 명세

> 이 문서는 Claude Code에 그대로 붙여넣어 작업을 지시하기 위한 **디자인 개편 브리프**다.
> 목표: 인덱스 페이지 + 모든 날짜별 브리핑을 "깔끔하고 전문성 있는" 하나의 디자인으로 **통일**한다.
> 대상 파일: `src/step5_assemble.py` (전부 여기서 렌더링됨), 산출물: `data/dashboard/*.html`, `data/dashboard/style.css`

---

## 0. 먼저 알아야 할 것 (진단)

작업 전에 아래 사실을 반드시 이해하고 시작할 것. 이걸 모르면 "7/9만 바꾸고 7/8은 그대로" 문제를 또 반복하게 된다.

1. **모든 페이지는 스타일시트 하나(`data/dashboard/style.css`)를 공유한다.**
   각 HTML(`index.html`, `2026-07-08.html`, `2026-07-09.html`, `alerts/*.html`)은 전부 `<link rel="stylesheet" href="style.css">`로 같은 파일을 참조한다. 이 CSS는 `step5_assemble.py`의 `_DASHBOARD_CSS` 상수에서 매 실행 시 다시 쓰인다.

2. **날짜별 페이지는 "정적 파일"이다.**
   `build_dashboard_html()`이 만든 `YYYY-MM-DD.html`은 그날 한 번 저장되고 이후 실행에서 다시 만들어지지 않는다(`main.py`가 같은 날 중복 실행을 `DuplicateRunError`로 건너뜀). 즉 **HTML 구조를 바꾸면 미래 페이지만 바뀌고, 이미 저장된 과거 페이지(7/8)는 그대로 남는다.**

3. **지금 7/8과 7/9의 차이는 "디자인"이 아니라 "데이터"다.**
   두 페이지는 똑같은 클래스(`card`, `meta` 등)와 똑같은 구조를 쓴다. 7/8은 기사 전부가 요약 실패(`summary_fallback`)라 제목 링크만 나열됐고, 7/9는 Gemini 요약과 `[확정]/[관측]` 태그가 채워졌을 뿐이다. **스크린샷의 카테고리 필터 칩·날짜 드롭다운·카드별 카테고리 배지·"요약 없음" 배지·"갱신" 타임스탬프는 현재 커밋된 코드에는 존재하지 않는다.** (누군가 그 페이지만 손으로/로컬에서 손댄 상태로 보인다.) → 이 명세는 그 요소들을 **정식 템플릿으로 코드에 편입**하는 것을 포함한다.

4. **핵심 레버 두 가지**
   - **CSS로 표현 가능한 변화는 공짜로 전체 통일된다.** `style.css`가 공유되고 과거·미래 페이지가 같은 클래스를 쓰므로, 색·타이포·여백·카드 스타일을 CSS에서 바꾸면 7/8까지 자동 반영된다(단, style.css가 다시 쓰여야 함 → §6-D).
   - **새 구조 요소(칩/필터/날짜 내비/헤더)는 CSS만으로 과거 페이지에 못 붙는다.** 과거 페이지를 **재생성(backfill)** 해야 한다 → §7.

## 1. 설계 원칙

- **단일 템플릿 · 단일 스타일시트.** 디자인은 오직 세 곳에서만 산다: `build_dashboard_html()`, `build_index_html()`, `_DASHBOARD_CSS`. 페이지별 인라인 스타일은 절대 만들지 않는다.
- **점진적 향상(Progressive Enhancement).** 자바스크립트는 필터 토글 정도의 바닐라 JS만, 그것도 없어도 전 기사가 보이도록. 빌드 스텝·프레임워크·유료 의존성 없음(GitHub Pages 정적 호스팅 유지).
- **보안 불변식 유지.** 렌더링 시 `_esc()`(HTML 이스케이프)와 `_safe_url()`(http/https만 허용) 호출을 **절대 우회하지 않는다.** RSS/뉴스 텍스트는 신뢰 불가 입력이다.
- **접근성 하한선.** 모바일까지 반응형, 키보드 포커스 링 유지, `prefers-reduced-motion` 존중, 색만으로 의미 전달 금지(태그는 색 + 텍스트 라벨 병행).

## 2. 디자인 토큰

방향성: **"정밀 계측기(precision instrument)".** 반도체 인텔리전스 제품답게 차갑고 정밀하고 신뢰감 있는 톤. 흔한 AI 기본값(크림+세리프+테라코타 / 블랙+형광 그린 / 신문 조판)은 피한다.

### 2-1. 색 (CSS 변수로 정의)

```css
:root {
  --paper:      #F5F7FA;  /* 배경: 차가운 연회색-블루 */
  --surface:    #FFFFFF;  /* 카드 표면 */
  --ink:        #10151C;  /* 본문/제목: 쿨톤 근흑 */
  --ink-soft:   #5A6472;  /* 메타·보조 텍스트 */
  --line:       #E3E8EF;  /* 헤어라인 보더 */
  --accent:     #1E5FBF;  /* 링크·활성 상태: 코발트 */
  --accent-ink: #16407F;  /* 링크 hover */
  --spectral-a: #1E5FBF;  /* 시그니처 그라디언트 */
  --spectral-b: #0E7C86;
  --spectral-c: #6D4AB5;

  /* 의미색: 확인 태그 */
  --confirmed:  #147B4B;  /* [확정] 초록 */
  --observed:   #B7791F;  /* [관측] 앰버 */
  --muted:      #6B7280;  /* 요약 없음 회색 */

  --warn-bg:    #FFF6E5;  /* 수집 상태 경고 행 */
  --warn-line:  #F0C36D;
}
```

### 2-2. 타이포그래피

- **본문·제목:** `Pretendard` — 무료·한글 최적 전문 산세리프. CDN 로드 + 시스템 폴백.
- **유틸리티/데이터(시그니처):** 모노스페이스 — 날짜·태그·소스명·표의 숫자·상태 타임스탬프에만 사용. 화면이 "계기판"처럼 읽히게 하는 유일한 개성 포인트.

```css
@import url("https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/dist/web/static/pretendard.css");

--font-sans: "Pretendard", -apple-system, "Segoe UI", "Apple SD Gothic Neo", sans-serif;
--font-mono: "JetBrains Mono", ui-monospace, SFMono-Regular, Menlo, monospace;
```

타입 스케일(모바일 우선, `rem`): h1 1.75 / h2 1.15(대문자 트래킹 살짝) / 카드 제목 1.05(700) / 본문 0.95(1.7 line-height) / 메타 0.8(mono).

### 2-3. 여백·모서리

간격 스케일 `4·8·12·16·24·32px`. 카드 radius `10px`, 칩 radius `999px`(pill), 보더는 전부 `1px solid var(--line)`. 컨테이너 max-width `820px`, 좌우 패딩 `20px`.

### 2-4. 시그니처 요소 (딱 하나)

마스트헤드 제목 아래 **얇은 스펙트럴 헤어라인**(2px, `--spectral-a → b → c` 저채도 그라디언트) — 실리콘 웨이퍼 간섭막의 무지갯빛을 은유. 페이지당 1회만. 나머지는 전부 조용하게.

```css
.masthead-rule {
  height: 2px; border: 0; margin: 12px 0 0;
  background: linear-gradient(90deg, var(--spectral-a), var(--spectral-b), var(--spectral-c));
  opacity: .7;
}
```

## 3. 컴포넌트 명세

### 3-1. 마스트헤드 (인덱스·데일리 공통)
- 제품명 `반도체 뉴스 데일리 브리핑` (h1) + 한 줄 설명 `반도체 업계 아침 인텔리전스` (mono, `--ink-soft`).
- 바로 아래 `.masthead-rule`.
- 데일리 페이지는 우측에 날짜 내비게이션(§3-6), 인덱스는 우측에 시스템 상태(§3-2).

### 3-2. 시스템 상태 배지 (기존 badge 대체)
- `● 최근 실행 성공` + 마지막 성공 시각(mono). 성공=초록 점, 실패=적색 점, 이력없음=회색 점.
- 큰 색 블록 대신 **작은 상태 점 + mono 텍스트**로 절제. `침묵 실패 방지` 원칙은 유지(실패 시 명확히 붉게).

### 3-3. 기사 카드 (`.card`)
구조(위→아래):
1. **칩 행:** 확인 태그 칩(§3-4) + 카테고리 칩(§3-5). 요약 실패 기사는 `요약 없음` 회색 칩.
2. **제목:** 700, 링크면 `--accent`.
3. **요약 본문:** 있으면 표시(`--ink`, line-height 1.7). 없으면 생략.
4. **메타 행(mono):** `소스명 · 원문 보기 ↗`.
- **좌측 4px 컬러 보더**로 태그 상태 색을 카드에 반영(확정=초록/관측=앰버/요약없음=회색). 색맹 대비 위해 칩 텍스트 라벨 병행.
- hover: 그림자 살짝(`0 2px 8px rgba(16,21,28,.06)`) + 보더 진하게. `transition: .15s`.

### 3-4. 확인 태그 칩 (`.tag`)
`.tag--confirmed`(확정), `.tag--observed`(관측), `.tag--muted`(요약 없음). pill, 배경은 의미색 8% 틴트, 텍스트는 의미색 진하게, mono, 대문자/괄호 표기 `[확정]`.

### 3-5. 카테고리 칩 (`.chip`)
`config/categories.yaml`의 분류명(메모리 / 장비·소재 / 팹리스·설계 / 규제·정책 등)을 pill로. 중립 회색, 아웃라인 스타일. 한 기사가 다중 카테고리면 여러 개 표시.

### 3-6. 날짜 내비게이션 (데일리 상단 우측)
- `‹ 이전`  `[YYYY-MM-DD ▾ select]`  `다음 ›` + `전체 목록`.
- `<select>`는 전체 날짜 목록으로 점프(값=`날짜.html`, `onchange`로 이동). 이전/다음은 정렬된 날짜 배열에서 계산해 앵커로. 데이터는 `build_dashboard_html`에 `all_dates: list[str]`를 넘겨 렌더.

### 3-7. 카테고리 필터 바 ("오늘의 핵심" 위)
- 스크린샷의 칩 필터. `전체` + 각 카테고리 토글 버튼.
- **바닐라 JS 점진적 향상:** 각 카드에 `data-categories="메모리 장비·소재"` 부여. 버튼 클릭 시 매칭 카드만 표시. **JS 미동작 시 전부 표시**(버튼은 그냥 링크처럼 무해). `aria-pressed` 토글, 키보드 접근 가능.

### 3-8. 수집 상태 표 (`table`)
- 숫자 열은 mono·우측 정렬. 경고 행(`today < avg7d*0.3`)은 `--warn-bg`/좌측 `--warn-line` 유지.
- 모바일에서 가로 스크롤(`overflow-x:auto`) 또는 라벨-값 스택.

### 3-9. 진행 중 이슈 / 속보 배너
- 이슈: 카드 형태 재사용, 엔티티는 카테고리 칩과 구분되는 아웃라인 칩. `first_seen ~ last_updated` mono.
- 속보 배너(`.alert-banner`): 적색 계열 유지하되 라운드·패딩·타이포를 새 토큰에 맞춰 정돈.

### 3-10. 푸터
- mono 한 줄: `자동 생성 · 소스: 삼성전자 뉴스룸 · SK하이닉스 · 디일렉 · EE Times · 전자신문 · ZDNet Korea`. `--ink-soft`, 상단 헤어라인.

## 4. 페이지 레이아웃

### 4-1. 인덱스 (`index.html`) — 첫 화면
현재 "제목 + 배지 + 링크 두 줄"뿐이라 가장 밋밋하다. 아래로 재구성:

```
┌───────────────────────────────────────────────┐
│  반도체 뉴스 데일리 브리핑        ● 정상 · 07-09 10:21│
│  반도체 업계 아침 인텔리전스                          │
│  ══════════ (스펙트럴 헤어라인) ══════════            │
│                                                 │
│  ┌── HERO: 최신 브리핑 ────────────────────────┐ │
│  │ 2026-07-09 (오늘)                    핵심 N건 │ │
│  │ 오늘 핵심 헤드라인 2~3개 미리보기 →           │ │
│  │                          [ 최신 브리핑 열기 ] │ │
│  └───────────────────────────────────────────┘ │
│                                                 │
│  지난 브리핑                                       │
│  ─ 2026-07-08  화                                │
│  ─ 2026-07-07  월                                │
│  … (날짜 + 요일, mono 정렬, hover 강조)            │
└───────────────────────────────────────────────┘
```
- HERO 카드: 최신 날짜 + (가능하면) 핵심 기사 수와 상위 헤드라인 미리보기 + 큰 CTA `최신 브리핑 열기`. 미리보기 데이터가 없으면 날짜·CTA만.
- 지난 브리핑: 밋밋한 `<ul>` 대신 날짜+요일 리스트(mono), 행 hover 강조. 항목이 많아지면 연/월 그룹핑 여지.

### 4-2. 데일리 (`YYYY-MM-DD.html`)
```
[‹이전]  [YYYY-MM-DD ▾]  [다음›]        (우측)
반도체 뉴스 데일리 브리핑
YYYY년 M월 D일 · HH:MM 갱신  (mono)
══════════ 스펙트럴 헤어라인 ══════════
[ 전체 | 메모리 | 장비·소재 | 팹리스·설계 | 규제·정책 ]   ← 필터 바
오늘의 핵심   → 카드 목록(칩+제목+요약+메타)
카테고리별     → 카테고리별 목록
확인 필요       → 목록
수집 상태       → 표
진행 중 이슈     → 카드 목록
푸터
```

## 6. 구현 지침 (`src/step5_assemble.py`)

### A. `build_dashboard_html()` 개편
- 시그니처: `all_dates: list[str] | None = None`, `updated_at: str | None = None` 인자 추가(날짜 내비·"갱신" 타임스탬프용).
- `<head>`에 마스트헤드/`viewport` 메타(`<meta name="viewport" content="width=device-width, initial-scale=1">`) 추가.
- 마스트헤드 + `.masthead-rule` + 날짜 내비(§3-6) + 필터 바(§3-7) 렌더.
- 각 기사 카드: 태그→`card--*`/`tag--*` 클래스 매핑, 카테고리→`data-categories` 속성과 `.chip`. **`_esc()`/`_safe_url()` 그대로 통과시킬 것.**
- "오늘의 핵심" 컨테이너에 `id="core"`(필터 스코프).
- 표는 `.table-wrap`으로 감싸고 숫자 셀 `class="num"`.
- 하단에 필터 JS + 푸터.

### B. `build_index_html()` 개편
- 마스트헤드 + 상태 배지를 `.status`(작은 점 + mono 시각)로.
- 최신 날짜를 `.hero` 카드로(§4-1). 가능하면 해당 날짜 핵심 헤드라인 미리보기 주입(없으면 CTA만).
- 지난 브리핑을 `.archive-list`로, 날짜 + 요일 표기.

### C. `build_alert_detail_html()` 도 새 마스트헤드/토큰에 맞춰 최소 정리(구조 동일, 클래스만 정합).

### D. **스타일시트가 반드시 다시 쓰이게 할 것**
`_DASHBOARD_CSS` 교체만으로는 부족하다 — 과거 페이지의 룩을 즉시 통일하려면 `data/dashboard/style.css`가 새 내용으로 덮여야 한다. 파이프라인을 새로 한 번 돌리면 `run()`이 style.css를 다시 쓴다. 데이터 없이 스타일만 갱신하려면 §7의 재생성 스크립트를 쓰거나, 최소한 새 `style.css`를 `data/dashboard/`에 수동 복사한다.

## 7. 과거 페이지 통일 (Backfill) — 가장 중요

CSS 변경은 §6-D로 전 페이지에 반영되지만, **새 구조 요소(칩/필터/날짜 내비/헤더 타임스탬프)는 이미 저장된 7/8 HTML에는 없다.** 과거 페이지를 새 템플릿으로 다시 렌더해야 한다.

`scripts/rebuild_dashboard.py`를 새로 만든다:

1. `data/summarized/*.json`(핵심 기사)과 `data/classified/*.json`("확인 필요")을 날짜별로 순회.
2. 각 날짜에 대해 `_compute_collection_stats`(로직 재사용)로 수집 통계 계산.
3. 같은 `build_dashboard_html(..., all_dates=<전체 날짜 목록>, updated_at=<파일 mtime 또는 기록값>)` 호출로 `YYYY-MM-DD.html` **재생성**.
4. 마지막에 `build_index_html`과 `style.css` 재작성.

```
python scripts/rebuild_dashboard.py         # 전체 재생성
python scripts/rebuild_dashboard.py --style-only   # style.css/index만 갱신(구조 변경 없이 룩만 통일)
```

**원천 데이터가 없을 때 처리(현재 레포는 `data/summarized/` 등이 비어 있음):**
- 원천 JSON이 로컬에 있으면 위 절차대로 완전 재생성.
- 없으면 최소한 `--style-only`로 CSS만 통일(과거 페이지도 새 색·타이포·카드 룩은 적용됨. 단 칩/필터/날짜 내비는 재수집 전까지 과거 페이지엔 안 붙음).
- 선택: `data/archive/YYYY-MM-DD.md`가 남아 있으면 마크다운을 파싱해 카드 데이터를 복원하는 폴백 파서도 가능(정확도 제한 있음, 필요 시에만).

> 권장: 앞으로는 `data/summarized/`, `data/classified/`를 삭제하지 말고 보관(재생성·재디자인 가능성 확보). `.gitignore` 정책과 별개로 로컬/아티팩트에 유지.
