# 실시간 트렌드 섹션 — 디자인·구현 명세 (인덱스 첫 화면)

> Claude Code에 붙여넣어 지시하기 위한 **추가 기능 브리프**다. `대시보드_디자인_개편_명세_v2_SAVE스타일.md`의 SAVE 스타일·토큰·통일 원칙을 그대로 따른다.
> 목표: 인덱스(첫 화면)에 **실시간 트렌드** 섹션을 추가한다. 왼쪽 **원형(도넛) 그래프**로 키워드별 비중(%)을, 오른쪽 **가로 막대그래프**로 같은 값을 **내림차순**으로 표시한다. 예: `삼성전자 42% · HBM 15% …`
> 대상 파일: `src/step5_assemble.py`(렌더링), 데이터 소스는 기존 config/state 재사용.

---

## 0. 배치 & 스코프

- **위치:** 인덱스 히어로 배너 바로 아래, 리포트 카드 리스트 **위**. "리포트와 같이" 한 화면에 보이는 섹션(별도 탭 아님). 탭 토글이 필요하면 §6-옵션 참고.
- **"실시간"의 의미:** 이 앱은 하루 한 번 도는 배치이므로, 트렌드는 **최신 브리핑(가장 최근 실행) 기준 스냅샷**이다. 섹션 부제에 `최신 브리핑 기준`을 달아 오해를 막는다. (롤링 N일 집계는 §7 확장.)
- 정적 사이트 원칙 유지: **서버사이드 인라인 SVG로 렌더**(빌드 스텝·JS 차트 라이브러리·유료 의존성 없음). Chart.js 등은 선택(§6-옵션).

## 1. 데이터 소스 & 계산 (기존 자산 재사용)

트렌드 키워드는 두 종류를 합쳐 하나의 순위로 만든다.

1. **기업/엔티티** — `config/company_aliases.yaml`의 정식명(별칭 매칭). 예: 삼성전자, SK하이닉스.
2. **토픽 키워드** — `config/keywords.yaml`의 `whitelist` 값들을 평탄화. 예: HBM, EUV, 파운드리 증설, 웨이퍼, 수출통제, 관세 등. (`blacklist`·`risk_keywords`는 집계에서 제외.)

### 집계 규칙 (`_compute_keyword_trends`)
- 입력: 최신 날짜의 기사 리스트(제목 + 요약 텍스트). 소스는 그날의 `data/summarized/*.json`(핵심) + `data/classified/*.json`(확인 필요)을 합친 것. 여의치 않으면 `issues.json`의 `entity`/`title`을 보조로 사용.
- 각 기사에 대해 각 추적어(기업 정식명·토픽 키워드)가 **본문(제목+요약)에 등장하면 그 추적어 카운트를 +1**(기사당 최대 1회 — 같은 기사 내 반복 언급은 중복 집계하지 않음). 기업은 별칭 중 하나라도 매칭되면 정식명으로 카운트.
- `total = 모든 추적어 카운트 합`. `pct_i = round(count_i / total * 100, 1)`.
- **내림차순 정렬** 후 **상위 N개(기본 8개)**만 개별 표시, 나머지는 `기타`로 합산해 마지막에 배치.
- 반환: `[{ "keyword": str, "count": int, "pct": float, "color": str }, ...]` (색은 §3-3 팔레트를 정렬 순서대로 배정, `기타`는 그레이 고정).

> 분모(100%)는 "추적어 총 언급 수"다. 한 기사가 여러 키워드에 걸리면 각각 카운트되므로 합이 기사 수보다 클 수 있다. 이 정의를 부제 툴팁이나 주석에 남길 것.

- 데스크톱/넓은 폭: **왼쪽 도넛(고정 ~160px) + 오른쪽 막대(가변)**, 세로 중앙 정렬.

## 3. 시각 스펙

### 3-1. 도넛 (원형그래프)
- 인라인 SVG, `viewBox="0 0 120 120"`, 링 하나에 세그먼트를 stroke로 그림. `cx=60 cy=60 r=45`, `stroke-width=18`, `fill=none`, 둥근 끝은 쓰지 않음(`stroke-linecap:butt`, 세그먼트 경계 또렷하게).
- 12시 방향에서 시작하도록 그룹을 `transform="rotate(-90 60 60)"`.
- **내림차순**으로 세그먼트 배치(큰 조각부터 시계방향).
- 가운데 구멍에 총 카운트 또는 최상위 키워드 % 표기(선택). 예: 굵은 숫자 + 작은 라벨.

### 3-2. 막대그래프 (내림차순)
- 각 행: `키워드 라벨` · `트랙(회색 pill) 안의 채움 막대(키워드 색, 폭=pct%)` · `우측 pct 값`.
- 순서는 도넛과 동일한 내림차순. 색도 도넛 세그먼트와 **1:1 일치**(막대가 곧 범례 역할 → 별도 범례 불필요).
- 막대는 순수 HTML/CSS(div 폭 %)로 — 반응형·경량.

### 3-3. 세그먼트 색 팔레트 (SAVE 톤)
정렬 순서대로 배정, `기타`는 마지막 그레이 고정.
```
["#6C5CE7", "#3D6FE6", "#1F7A6B", "#C2652A",
 "#A9790B", "#3B8B4E", "#B5457E", "#4E7CA1"]   // 상위 8색
기타: "#B8BEC8"
```

## 4. 도넛 SVG 수학 (구현자용)

원둘레 `C = 2·π·r = 2·π·45 ≈ 282.743`.
정렬된 세그먼트를 순회하며 누적 비율 `cum`(이전까지 합, 0~1)을 유지한다. 세그먼트 `i`(비율 `f_i = pct_i/100`):

```
seg_len   = f_i * C
gap_len   = C - seg_len
dasharray = f"{seg_len:.3f} {gap_len:.3f}"
dashoffset = -(cum * C)          # 음수로 시작점을 앞으로 이동
cum += f_i
```
각 세그먼트는 아래 `<circle>`로 렌더(그룹에 rotate 적용):
```html
<circle cx="60" cy="60" r="45" fill="none"
        stroke="{color}" stroke-width="18"
        stroke-dasharray="{dasharray}" stroke-dashoffset="{dashoffset}"/>
```

**워크드 예시** (삼성 42 / SK 18 / HBM 15 / EUV 9 / 파운드리 6 / 수출통제 4 / 기타 6):
- 삼성전자: seg=0.42·282.743≈**118.75**, offset **0**
- SK하이닉스: seg≈**50.89**, offset=-(0.42·C)≈**-118.75**
- HBM: seg≈**42.41**, offset=-(0.60·C)≈**-169.65**
- … 이후 동일하게 누적.

## 5. 반응형·접근성
- 모바일에서 도넛 축소(예 132px)·막대 전체폭. `@media(max-width:480px)`로 스택 전환.
- 색만으로 구분되지 않도록 **막대·도넛 모두 키워드 텍스트 라벨과 % 병기**.
- 도넛 SVG에 `role="img"`와 `<title>`(예: "실시간 트렌드: 삼성전자 42% 등") 부여.
- `prefers-reduced-motion` 존중(애니메이션 넣을 경우).

## 6. 구현 지침 (`src/step5_assemble.py`)

**A.** `_compute_keyword_trends(articles, *, top_n=8) -> list[dict]` 추가 — §1 규칙. `company_aliases.yaml`·`keywords.yaml`을 로드(이미 다른 스텝에서 파싱하면 재사용). 매칭은 `_esc` 이전의 원문 텍스트로 하되, **출력 라벨은 반드시 `_esc()`로 이스케이프**.

**B.** `render_trend_section(trends) -> str` 추가 — 도넛 SVG(§4) + 막대 HTML(§3-2)을 카드로 조립해 문자열 반환. 데이터가 비면(`total==0`) 섹션을 생략하거나 "데이터 없음" 안내.

**C.** `build_index_html(...)`에서 히어로 아래·리포트 리스트 위에 `render_trend_section(...)` 삽입. 트렌드는 **최신 날짜 기사**로 계산.

**D.** 색 배정은 정렬 인덱스 기준(§3-3), `기타`는 그레이. 도넛 세그먼트와 막대 색이 동일 키워드에서 반드시 일치.

**보안:** 키워드 라벨은 config에서 오지만 방어적으로 `_esc()` 적용. SVG 수치는 float 포맷으로만 생성(문자열 삽입 금지).

**옵션 (탭/라이브러리):**
- 인덱스를 `리포트`/`실시간 트렌드` 두 탭으로 나누고 싶으면 버튼 토글 + 섹션 show/hide(바닐라 JS, JS 없으면 둘 다 표시). 기본은 "같이 보이는 섹션".
- Chart.js CDN을 쓰면 애니메이션·툴팁이 쉬우나 JS 의존이 생긴다. 정적·경량 원칙상 **기본은 서버사이드 SVG**를 쓰고, 라이브러리는 선택.

## 7. 참조 구현 스니펫

### 7-1. 집계(파이썬)
```python
def _compute_keyword_trends(articles, top_n=8):
    terms = _load_tracked_terms()   # {정식명: [별칭...], 토픽키워드: [자기자신]}
    counts = {}
    for art in articles:
        text = f"{art.get('title','')} {art.get('summary','')}"
        for canonical, aliases in terms.items():
            if any(a in text for a in aliases):
                counts[canonical] = counts.get(canonical, 0) + 1
    total = sum(counts.values())
    if total == 0:
        return []
    ranked = sorted(counts.items(), key=lambda kv: kv[1], reverse=True)
    top = ranked[:top_n]
    rest = sum(c for _, c in ranked[top_n:])
    palette = ["#6C5CE7","#3D6FE6","#1F7A6B","#C2652A","#A9790B","#3B8B4E","#B5457E","#4E7CA1"]
    out = [{"keyword": k, "count": c, "pct": round(c/total*100, 1), "color": palette[i % len(palette)]}
           for i, (k, c) in enumerate(top)]
    if rest:
        out.append({"keyword": "기타", "count": rest, "pct": round(rest/total*100, 1), "color": "#B8BEC8"})
    return out
```

### 7-2. 도넛 SVG(파이썬)
```python
import math
def _donut_svg(trends, total_count):
    C = 2 * math.pi * 45
    cum, segs = 0.0, []
    for t in trends:
        f = t["pct"] / 100.0
        seg = f * C
        segs.append(f'<circle cx="60" cy="60" r="45" fill="none" stroke="{t["color"]}" '
                    f'stroke-width="18" stroke-dasharray="{seg:.3f} {C-seg:.3f}" '
                    f'stroke-dashoffset="{-(cum*C):.3f}"/>')
        cum += f
    ring = "".join(segs)
    return (f'<svg viewBox="0 0 120 120" role="img" width="160" height="160" class="donut">'
            f'<title>실시간 트렌드</title>'
            f'<g transform="rotate(-90 60 60)">{ring}</g>'
            f'<text x="60" y="58" text-anchor="middle" class="donut-num">{total_count}</text>'
            f'<text x="60" y="72" text-anchor="middle" class="donut-cap">건</text>'
            f'</svg>')
```

### 7-3. 막대 + 카드 HTML
```python
def render_trend_section(trends, total_count):
    if not trends:
        return ""
    bars = "".join(
        f'<div class="tbar"><span class="k">{_esc(t["keyword"])}</span>'
        f'<span class="track"><span class="fill" style="width:{t["pct"]}%;background:{t["color"]}"></span></span>'
        f'<span class="p">{t["pct"]}%</span></div>'
        for t in trends
    )
    return (f'<section class="trend"><div class="trend-head">'
            f'<h2 class="sec">실시간 트렌드</h2><span class="sub">최신 브리핑 기준</span></div>'
            f'<div class="trend-body"><div class="trend-donut">{_donut_svg(trends, total_count)}</div>'
            f'<div class="trend-bars">{bars}</div></div></section>')
```

## 8. 백필·재생성
트렌드는 **인덱스에서 최신 날짜 데이터로만** 계산하므로, `scripts/rebuild_dashboard.py`가 `build_index_html`을 다시 부를 때 자연히 갱신된다(별도 과거 페이지 수정 불필요). 최신 날짜의 `data/summarized/`·`data/classified/` JSON이 있어야 정확히 계산되며, 없으면 `issues.json`의 엔티티로 축약 계산하거나 섹션을 생략한다.
